General Information

Some steps of this guide are only necessary for Android or iOS development. In those cases, the step is marked with the corresponding icon.

  • for Android development
  • for iOS development
  • for development on Windows 10

General Prerequisites

For this course you need an AR compatible device and an USB-Cable which connects your device to your laptop.

  • If you have an iOS-Device you need macOS to deploy the project to the device.
  • XCode 10.3 or 9 (infos later)
  • XCode Commandline Tools (infos later)

Compatible Devices

  • Android
  • iOS

Install Unity

  1. Download + Install Unity Hub

  2. Login + Activate License (if necessary)​

  3. Go to Installs
  4. Select Add, Unity 2019.2.2f1
    • Set checkmarks: Android Build Support, Android SDK, OpendJDK
    • Set checkmark: iOS build support
    • Set checkmark: Visual Studio Community, if no you do not already have Visual Studio installed

iOS Toolchain

​1. Download and install XCode 10.3

  1. Install XCode Toolchain

Install Workshop Files

  1. Clone the project or download the ZIP of the Project Files in your desired folder

    Note

    Example: "C:\Users\USERNAME\Development\arworkshop" | Now Called: "PROJECT_FOLDER"​

Verify the Installation

  1. Open Unity Hub

  2. Choose Add and select the folder of the Hello-World project

    Important

    Unity Hub cannot detect if your project is in a subfolder, so make sure to select the right one. E.g.: "PROJECT_FOLDER\Hello-World".

  3. Open up the Build Settings (cmd + shift + b or via File > Build Settings)

    1. Make sure that you have the open scene SampleScene under Scenes In Build
    2. Switch the Platform to your device type (Anroid or iOS)

    Error

    add print screen, (need final unity project)

    1. Open the Player Settings...

      1. Open the Other Settings menu
      2. Check that the field Camera Usage Description is not empty
      3. Under Target minimum iOS Version insert the value 11.0
      4. Select ARM64 from the Architecture drop down
      5. Close the window

  4. Ensure your device has Android Debugging enabled

    1. Plug-in the device on your Laptop

    2. Verify that the device is reachable by adb Ender adb device -l in you command line. You should now see something like:

      8ATX0YYJK device product:blueline model:Pixel_3 device:blueline transport_id:1

    3. If the device is shown as unauthorized

    Go to the developer options on your mobile device and click "Revoke USB debugging authorization" Restart ADB: By entering adb kill-server followed by adb start-server in your command line

    1. Reconnect your device and accept the connection on your device screen
    2. Reverify according to step II.

  5. Open the Window > Package Manager

    1. Check that you have the version preview.1 - 3.0.0 of ARFoundation, upgrade if necessary (click on the arrow on the left and then on See all versions)
    2. Check that the preview.1 - 3.0.0 ARKit XR Plugin is installed
    3. Check that the preview.1 - 3.0.0 ARCore is installed

  6. Go the the Build Settings again (cmd + shift + b) and hit Build and Run

  7. Create a folder called Builds in the project folder (the project folder contains the Assets, Library, Logs, ProjectSettings folders). Give your build the name that you like.

  8. After Unity has finished building, Xcode will start. Select your iphone from the active scheme.

    If you encounter any issues please follow the steps below:

    1. Make sure you have a developer account listed under Apple IDs in Xcode > Preferences > Accounts

      Close the Preference window

    2. Check Automatically manage signin under Targets > Unity-iPhone > Signing and select the Team

      You may be asked to allow codesign to access your keychain

    3. Open the Keychain Access application, go to your login Keychain and select the Certificate category. Search for an iPhone Developer certificate and copy its Organizational Unit.

    4. Go back to your the Unity Player Settings and paste the copied value in the field Signing Team ID which is located under Other Settings

    5. Close the Player Settings and Build And Run your project again.

  9. Upon completed build, Xcode asks to unlock your iPhone

  10. Trust the developer under Settings > General > Device Management > Developer App > Trust "<insert your email here>" > Trust

  11. If the app is not automatically started select it from your home screen

  12. Validate app functions

    When the app is running it should detect surfaces of the real world. When tapping the screen a new cube is spawned, which bounces of the surfaces.
